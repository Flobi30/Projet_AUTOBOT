## P3 — Dispatch Async (2026-04-02)

### Architecture implemented
```
WebSocket
    ↓
RingBufferDispatcher (P2) — per-pair RingBuffer
    ↓  one reader per pair (not per instance!)
AsyncDispatcher (P3)
    ↓  _dispatch_loop task per pair
    ↓  O(1) dict lookup: pair → [InstanceQueue]
    ↓  put_nowait_drop_oldest (never awaits)
InstanceQueue (1 per instance)
    ↓  await queue.get()
TradingInstanceAsync._queue_consumer_loop()
```

### Files created
- `async_dispatcher.py` — AsyncDispatcher: 1 reader/pair → N queues/instances
- `instance_queue.py` — InstanceQueue: asyncio.Queue + drop-oldest backpressure
- `tests/test_async_dispatcher.py` — 39 tests (unit + lifecycle + dispatch + e2e + perf)
- `tests/test_instance_queue.py` — 29 tests (correctness + perf + integration)

### Files modified
- `instance_async.py` — Added: attach_queue(), start_queue_consumer(), _queue_consumer_loop(), stop() drain
- `orchestrator_async.py` — Integrated AsyncDispatcher (P3 replaces ring_dispatcher.run_consumer)

### Key properties verified by tests
- O(1) dispatch: pair → List[InstanceQueue] dict lookup
- Non-blocking: put_nowait_drop_oldest never awaits
- Backpressure: drop-oldest policy, metrics visible (drop_count, fill_ratio)
- Fan-out: N instances on same pair all receive the tick
- Isolation: XBT/EUR ticks never reach ETH/EUR queues
- Graceful shutdown: drain() + CancelledError handled correctly
- Backward compat: attach_queue() optional — old on_price_update path still works
