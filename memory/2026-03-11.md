## 🚨 NOUVEAU PROCESSUS (important)

**Règle dorée : PLUS DE PUSH SANS VALIDATION DUALE**

Avant chaque push sur master :
1. **Gemini review** → check thread-safety, logique, performance
2. **Opus review** → check sécurité, edge cases, architecture
3. **Correction** des problèmes identifiés
4. **Re-review** si changements significatifs
5. **Push** uniquement après validation des deux

**Pourquoi ?** 
- J'ai poussé du code avec des warnings non corrigés
- Risque de régressions en production
- Trading = argent réel, zero tolérance aux erreurs
